# Rubric

Use this project rubric to understand and assess the project criteria.

## Agent Architecture & Patterns

### Criteria

Demonstrate the ability to implement an iterative refinement workflow using the generator-validator pattern.

### Submission Requirements

Agent Instances Created: The submission must successfully create a ResearcherAgent instance and a ResearchCriticAgent instance, both initialized with the correct model parameter.

LoopAgent Composition: A LoopAgent object must be instantiated with the name "research_refinement_loop". The sub_agents list must contain [researcher, critic] in that specific order. The max_iterations parameter must be configured.

Workflow Execution: When the code runs, the console output must show "Created LoopAgent: research_refinement_loop" and "Type: LoopAgent". The logs must display multiple iterations (Researcher → Critic cycles) with quality scores for each iteration.

### Criteria

Implement a concurrent workflow using parallel agents

### Submission Requirements

ParallelAgent for Fan-out: A ParallelAgent named "parallel_source_searches" must be created. Its sub_agents list must include all three search agents: [web_search, arxiv_search, scholar_search].

SequentialAgent for Orchestration: A SequentialAgent named "source_gathering_workflow" must be created to wrap the parallel process. Its sub_agents list must be [parallel_searches, aggregator].

Concurrent Execution: The console output must show "Type: ParallelAgent". Critically, all three search agents must appear to run in the logs before the aggregation step occurs (Fan-out pattern).

### Criteria

Configure a deterministic agent router.

### Submission Requirements

LlmAgent Initialization: In `agents/router.py`, the super().**init()** call must include the correct parameters: name ("domain_classifier"), model, and instruction.

Routing Configuration: The GenerateContentConfig must use a low temperature (e.g., temperature=0.3) to ensure deterministic behavior. The response_mime_type must be set to "application/json".

Classification Output: The console output must display "Domain: [classified_domain]" and a "Confidence" score. Running the same query multiple times should produce consistent domain classification.

## Workflow Integration & Optimization

### Criteria

Compose a complex, multi-stage workflow

### Submission Requirements

Stage Execution: In agents/orchestrator.py, the functions execute_source_gathering() and execute_research_loop() must be called.

Data Flow: Data must pass correctly between stages (e.g., Sources → Research Loop → Fact Check).

Complete Workflow: The console must show the completion of all 7 stages: Classification, Source Gathering, Research, Fact Check, Synthesis, Citations, and Performance Evaluation.

Report Generation: The console must output "All 7 stages executed successfully" and generate a research_report.md file.

### Criteria

Implement a monitoring system that tracks metrics

### Submission Requirements

Performance Evaluator Instantiation: A PerformanceEvaluator instance must be created in agents/orchestrator.py, and the total execution_time must be calculated.

Metrics Recording: The method evaluate_query_result() must be called with a dictionary containing: quality_score, sources_found, iterations, fact_checks, and citations_count.

Results Integration: A performance summary (containing health status and score) must be generated via analyze_performance() and added to the final workflow_results. The console output must display these metrics at the end of the run.

## Agentic System Execution

### Criteria

Implement Google Agent Development Kit agent objects

### Submission Requirements

Code Execution: Running python main.py must complete the full workflow without raising Python exceptions (excluding valid authentication errors).

ADK Agent Objects: The console logs must verify the types of agents created: "Type: LoopAgent", "Type: ParallelAgent", "Type: SequentialAgent". These must not be NoneType.

Pattern Adherence: The implementation must use the provided ADK classes (LlmAgent, LoopAgent, etc.) rather than manually writing Python while loops or if statements to simulate agents.

### Criteria

Generate a report against a custom query using Agentic Workflows

### Submission Requirements

Generated Report: A research_report.md file must be submitted containing populated sections for: Executive Summary, Research Findings, Key Insights, Bibliography, and Methodology.

Custom Research Query: The README.md must document a custom query used by the student (e.g., "How do mRNA vaccines work?"). The submitted report must match this query, not the default "Quantum Computing" example.

Documentation: The README.md includes observations about the workflow (e.g., how many iterations it took).
