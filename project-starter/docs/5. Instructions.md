# Project Instructions

## The Engineering Backlog

Complete the code in the `agents/` directory to build your research assistant. Look for the `TODO` markers in the code.

### Task 1

**Context**: Our current model writes one draft and stops. It often misses the nuance.
**Task**: Implement a **LoopAgent** pattern in `agents/researcher.py`.

### Task 2

**Context**: Searching sources sequentially takes too long. Gather intel from everywhere at once.
**Task**: Implement a **ParallelAgent** workflow in `agents/source_gatherer.py`.

### Task 3

**Context**: A physics question needs different handling than a history question.
**Task**: Configure the **LlmAgent** in `agents/router.py`.

### Task 4

**Context**: The components (Router, Searchers, Editor) are disconnected.
**Task**: Wire the system together in `agents/orchestrator.py`.

- Requirements:
  1. In `execute_research_workflow`, connect Stage 2 (Source Gathering) to Stage 3 (Research Loop).
  2. Pass the data from the gathered sources into the research loop.
  3. Ensure the workflow handles asynchronous execution (`await`) correctly.

### Task 5

**Context**: You don't know if your improvements are actually working. Implement metrics.
**Task**: Implement the **PerformanceEvaluator** in `agents/orchestrator.py`.

## Verification & Submission

### How to Test

Run the main application:

```sh
python main.py
```

### Success Criteria

- The console should show the creation of `LoopAgent`, `ParallelAgent`, and `SequentialAgent` objects.
- You should see the "Thought Process" in the logs: The Router classifying, the Searchers running in parallel, and the Researcher refining drafts.
- A `research_report.md` file should be generated in your project folder.

## Submission Instructions

### Deliverables

To complete this project, submit a ZIP file containing:

1. **Code**: Your modified `agents/` directory with all TODOs completed.
2. **Artifacts**: The generated `research_report.md` and screenshots of your running research assistant.
3. **Documentation**: The `README.md` updated with a custom research query you ran and your observations on the agent's performance.

Follow these steps to prepare your submission.

#### Step 1: Do not submit the default query ("Quantum Computing")

- **Choose a unique topic**. It could be "The impact of CRISPR on agriculture," "The history of the Rust programming language," or "The efficacy of remote work on productivity."
- **Run the custom query**.
- **Verify the Output**. Ensure a new research_report.md is generated with content specific to your topic.

#### Step 2: Capture Evidence (Screenshots)

We need visual confirmation that your agents are orchestrating correctly in real-time.

**Take screenshots of your CLI / Terminal** showing the following events:

- **Initialization**: The console showing "Created LoopAgent", "Created ParallelAgent", etc.
- **The Logic Loop**: The "Thought Process" logs where the Router classifies the domain, and the Researcher/Critic loop iterates (e.g., "Quality below threshold - Continue refining...").
- **Completion**: The final "WORKFLOW COMPLETED" summary showing the stats (Iterations, Sources Found, Credibility Score).

#### Step 3: Rubric Check

Before zipping your files,check the rubric on the following pages to check for completion of all of the rubric items.

#### Step 4: Package & Submit

Gather the following files from your workspace (right-click files in the explorer to download them):

**1. The Code:**

- `agents/researcher.py`
- `agents/source_gatherer.py`
- `agents/router.py`
- `agents/orchestrator.py`
- `main.py` (if you modified it)

**2. The Artifacts:**

- `research_report.md` (The report generated from your unique query).
- **CLI Screenshots** (Images showing your terminal output).

**3. The Documentation:**

- `README.md`: Update this file with your **Unique Research Query** and a brief note on what you observed during the run.

**Final Step**: Zip these files together and submit the zip archive.
