# Starter Code and Workspace

## How to Use the Starter Code

You are inheriting a "skeleton" application. The infrastructure (logging, configuration, API clients) is built, but the agent logic is missing.

You can find the starter code in the workspace below.

Your job is to locate the **`TODO`** markers within the `agents/` directory and implement the ADK and workflow patterns

1. Authenticate: The system needs a valid ID to talk to Vertex AI.
   - Rename `.env.example` to `.env`.
   - Edit the file according to the "Configuring Vertex AI Service Account" instructions on the previous page.
2. Smoke Test: Run `python main.py`. It _will_ fail with errors like `AttributeError: 'NoneType' object...`.

**This is good**. It means the infrastructure works, but your agents are empty (returning None).

Once your TODOs are complete, you run the full system to verify the Research Assistant.

Run the Default Suite:

```sh
python main.py
```

If successful, your code will generate `research_report.md`. Open this file. If it contains a synthesized answer with citations and a bibliography, you have successfully engineered an autonomous research agent.
